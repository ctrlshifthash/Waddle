## ❄️ What is Waddle World?

**Waddle World** is an original, browser-based multiplayer game inspired by the classic snowy-island social games — reimagined with a real **play-to-earn** layer on **Solana**. Make a penguin, waddle around shared rooms with players from everywhere, chat, play **9 arcade minigames**, battle in **Card-Jitsu**, adopt **puffles**, decorate your **igloo**, climb the ranks… and **claim real SOL** straight to your wallet from a shared daily pool.

It's **free to play** — no purchase, no wallet required to start. Connect a Solana wallet only when you want to claim your rewards. Everything runs in the browser; the whole island, every penguin, and all the furniture are rendered **procedurally**, so it loads instantly.

> 🪂 **The hook:** a fixed pool of **1 SOL is shared across *all* players every single day**, paid out in real SOL — first come, spread evenly, hard-capped, and fully server-authoritative.

## ✨ Features

### 🌍 World & Social
- **Multiplayer worlds** — create or join any world by name; each is its own island. A live browser shows active worlds + player counts.
- **11 themed rooms + your own igloo** — Town, Coffee Shop, Night Club, Plaza, Pizza Parlor, Pet Shop, Dock, Beach, Snow Forts, Dojo, Ski Village — each with snow / water / night / interior scenery.
- **Click-to-waddle movement** — smooth, networked, with facing + walk animation.
- **Chat** — free text (profanity-filtered), preset **Safe Chat**, and **emotes** as speech bubbles.
- **Player cards & friends** — click any penguin to see their level, belt, stamps and outfit, then add them as a friend and **jump to them** in any room.
- **Instant map travel** — a 🗺️ menu to hop anywhere, plus walk-through doorways.

### 🎮 9 Minigames
All payouts are **server-validated** with anti-spam caps — you earn coins by actually playing.

### 🥋 Card-Jitsu
The classic elemental card battle — 🔥 beats ❄️ beats 💧 beats 🔥 — at the **Dojo**, with full **ninja-belt progression** (White → … → Black → Ninja).

- **Puffles** — adopt colored pets at the Pet Shop; your active puffle waddles around with you and everyone can see it.

- **Levels & rank** — earning coins grants XP and ranks you up (Newcomer → Legend), with bonus coins on every level-up.

- **Shop & Closet** — buy colors, clothing and furniture (each with distinct in-game art) and equip them with a live penguin preview that everyone sees instantly.

## 🪂 How the Daily SOL Pool Works

The headline P2E mechanic — designed to be **fair, abuse-resistant, and impossible to overspend**:

| Rule | Value |
|------|-------|
| 💧 **Shared daily budget** | **1 SOL** across *all* players, per UTC day |
| 🎯 **Per claim** | 0.002 SOL, sent as **native SOL** to your wallet |
| 🔁 **Claims per player / day** | up to **3** |
| ⏳ **Release schedule** | **time-vested** — the pool unlocks gradually over the day so early players *and* later joiners can claim |
| 🛡️ **Hard cap** | the server will **never** pay out more than 1 SOL in any 24h, full stop |

**Server-authoritative & safe by construction:** every claim is validated on the Colyseus server, which tracks the day's spend in persistent storage, enforces the per-player cap, vests the budget by the fraction of the day elapsed, and uses an in-flight guard to prevent double-claims. The treasury key lives **only** on the server and signs the transfers — it's never exposed to the browser. Real payouts are gated behind `SOL_FAUCET_ENABLED` + a funded treasury; with them off, the UI honestly shows rewards as "not live yet."

## 🧱 Tech Stack

| Layer | Technology |
|-------|-----------|
| **Multiplayer server** | [Colyseus](https://colyseus.io) 0.16 — authoritative rooms + schema state sync over WebSocket |
| **Game rendering** | [Phaser](https://phaser.io) 3.90 — fully procedural penguins, rooms & furniture |
| **App shell / UI** | React 18 + Vite 6 |
| **Wallet & auth** | [Privy](https://privy.io) — embedded + external Solana wallets |

| **Language** | TypeScript everywhere · ESM · npm workspaces |
| **Hosting** | [Railway](https://railway.app) — single service hosting site + API + socket |

## 🏗️ Architecture

Waddle World ships as **one deployable service** with three TypeScript workspaces:

```
  Browser (React + Phaser)  ⇄  Colyseus Server (authoritative)  ⇄  Solana
        │                              │
        └── same-origin WebSocket ─────┘
            (the server also serves the built site)
```

- **One Colyseus room type**, `world`, partitioned by an `area` option (`town`, `dock`, `igloo:<key>`, …). Players sharing an `area` share a room instance via `filterBy(["area"])`.
- **Synced schema state** carries positions, appearance, chat bubbles and igloo furniture; one-off events (coin awards, claim results, notices) are sent as typed messages.
- **`GameClient` is the single bridge** on the client — it owns the Colyseus connection and emits typed events that both **React** (HUD, modals, dashboard) and the **Phaser scene** (rendering) subscribe to.
- **Server-authoritative economy** — all coin awards, minigame scores and SOL claims are validated server-side for P2E integrity.
- **Single-service deploy** — the server serves the built client (`client/dist`) as static files with an SPA fallback, so the website, the REST API (`/worlds`, `/health`), and the realtime socket all live on **one domain**. The client auto-connects its socket to its own origin — zero cross-service URL wiring.

## 📁 Project Structure

```
shared/                    # contracts shared by client + server (zero deps)
  protocol.ts              #   message names, payloads, JoinOptions
  rooms.ts                 #   room layouts, doors, features, igloo def
  items.ts                 #   clothing + furniture catalogs
  economy.ts               #   coin amounts, minigame payouts, SOL pool rules
  safechat.ts              #   safe-chat phrases, emotes, profanity filter

server/                    # Colyseus game server (also serves the site)
  src/index.ts             #   bootstrap: express + ws transport + static client
  src/rooms/WorldRoom.ts   #   the room: movement, chat, shop, igloo, SOL claims
  src/schema/state.ts      #   @colyseus/schema synced state
  src/services/            #   store (lowdb), auth (Privy verify), solana (payouts)

client/                    # React + Phaser + Privy
  src/net/GameClient.ts    #   the Colyseus ⇄ React ⇄ Phaser bridge
  src/net/endpoint.ts      #   same-origin / configurable socket URL resolver
  src/game/                #   Phaser scenes, penguin rig, procedural gfx
  src/game/minigames/      #   the 9 arcade minigames
  src/ui/                  #   HUD, chat, shop, closet, wallet, dashboard, igloo editor
  src/ui/Landing.tsx       #   marketing landing page
```

## 🚀 Quick Start (Local Dev)

```bash
npm install
npm run dev
```

Then open **http://localhost:5173**.

- Starts the Colyseus server (`:2567`) and the Vite client (`:5173`) together.
- **No `.env` needed to play** — you join as a **Guest** and coins are tracked locally.
- Open a second browser tab to see real-time multiplayer (two penguins, live).

```bash
npm run dev           # server + client together
npm run dev:server    # just the Colyseus server
npm run dev:client    # just the Vite client
npm run build         # production build of the client
npm --workspace server run typecheck
npm --workspace client run typecheck
```

## 🌐 Deployment

Waddle World runs as a **single service** — the game server builds and serves the website too. Live on **[Railway](https://railway.app)** with a persistent volume for save data.

**Build command**
```bash
npm install --include=dev && npm run build --workspace client
```
**Start command**
```bash
npm run start --workspace server
```

- Attach a **persistent volume** and point `DATA_DIR` at its mount path (e.g. `/data`) so player progress + the daily SOL cap survive restarts.
- A [`render.yaml`](./render.yaml) blueprint is included for one-click deploys on Render too.
- Add your live domain to **Privy → Allowed origins**, and **fund the treasury wallet** before enabling real payouts.

## ⚙️ Configuration

A single `.env` at the repo root serves both client and server (all kept out of git):

```ini
# --- Client (public, baked into the build) ---
VITE_PRIVY_APP_ID=your-privy-app-id        # from https://dashboard.privy.io
# VITE_GAME_WS_URL is optional — unset = connect to the same origin (single-service)

# --- Persistence ---
DATA_DIR=/data                              # mount a volume here for durable saves

# --- Daily SOL pool (real payouts) ---
SOL_FAUCET_ENABLED=true
SOLANA_RPC_URL=https://your-rpc-endpoint    # e.g. a Helius mainnet URL
SOLANA_TREASURY_SECRET=<base58 secret of the FUNDED treasury wallet>

# --- Server auth (optional) ---
PRIVY_APP_ID=...
PRIVY_APP_SECRET=...
PRIVY_ENFORCE=false                         # require a verified Privy token to join

# --- Coin → SPL settlement (future season; off by default) ---
SOLANA_SETTLE_ENABLED=false
SOLANA_PAYOUT_MODE=mint
SOLANA_COIN_MINT=
```
